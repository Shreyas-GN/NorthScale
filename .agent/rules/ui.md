# UI Rules

# Core UI Philosophy

This application is an:

> AI-native institutional investment terminal.

The interface must feel:

* premium
* futuristic
* calm
* intelligent
* cinematic
* layered
* narrative-first

NOT:

* dashboard-heavy
* crypto-style
* cluttered
* spreadsheet-like
* retail-trader focused

---

# Design System Rules

The UI must follow:

* DESIGN.md strictly
* spacing consistency
* typography hierarchy
* dark-first styling
* layered surfaces
* subtle motion system

Never improvise visual styles outside the design system.

---

# Layout Rules

Global application structure:

```txt id="uimainlayout"}
LEFT SIDEBAR
MAIN CONTENT
RIGHT INTELLIGENCE PANEL
```

This structure should remain consistent across:

* dashboard
* stock pages
* portfolio views
* AI interfaces

---

# Sidebar Rules

The sidebar must:

* remain minimal
* support keyboard navigation
* use icon-first navigation
* feel floating and layered

Sidebar contains:

* Portfolio
* Watchlists
* Alerts
* AI Queries
* Sectors
* Settings

Avoid:

* oversized navigation
* excessive nesting
* clutter

---

# Main Content Rules

The main content area prioritizes:

1. recommendation
2. conviction
3. AI thesis
4. key risks
5. supporting signals

ONLY THEN:

* charts
* detailed metrics
* raw financial tables

Narrative-first UX is mandatory.

---

# Right Intelligence Panel Rules

The right panel should feel:

> "alive but calm."

Contains:

* AI insights
* ownership changes
* valuation shifts
* risk alerts
* earnings observations

Avoid:

* noisy notifications
* excessive colors
* flashing updates

---

# Recommendation Card Rules

The recommendation card is the HERO component.

It must contain:

* BUY/HOLD/SELL state
* conviction score
* AI thesis summary
* confidence level
* key supporting signals

Visual style:

* subtle glow
* elevated depth
* premium gradients
* layered shadows

Avoid:

* casino green/red visuals
* oversized badges
* meme-stock aesthetics

---

# Card System Rules

Cards must:

* float subtly
* use soft depth
* remain readable
* use subtle borders

Avoid:

* flat enterprise cards
* dense spreadsheet layouts
* hard borders everywhere

---

# Typography Rules

Use:

* Geist Sans
* Geist Mono

Typography must feel:

* precise
* institutional
* readable
* calm

Avoid:

* giant bold textt
* excessive uppercase
* cramped spacing
* tiny finance text

---

# Motion Rules

Allowed:

* fade transitions
* blur transitions
* hover elevation
* subtle scaling
* smooth tab transitions

Forbidden:

* bouncing
* flashy animations
* excessive parallax
* gaming motion
* crypto-casino effects

Animation duration:
120ms–220ms

---

# Color Rules

Primary themes:

* graphite
* indigo
* muted cyan
* soft emerald

Avoid:

* rainbow gradients
* neon overload
* oversaturated palettes

Use semantic colors carefully:

* green for BUY
* yellow for HOLD
* red for SELL

Never let semantic colors dominate the interface.

---

# Chart Rules

Charts are SECONDARY.

Charts must:

* remain minimal
* use muted colors
* avoid clutter
* support readability

Avoid:

* TradingView overload
* excessive indicators
* rainbow technical analysis

---

# Table Rules

Tables must:

* remain breathable
* use larger row spacing
* minimize borders
* support sticky headers

Avoid:

* Excel-style density
* tiny financial rows
* overloaded columns

---

# Command Palette Rules

The command interface is critical.

Inspired by:

* Raycast
* Cursor

Must support:

* instant search
* keyboard-first UX
* AI-driven navigation
* stock analysis commands

The command palette should feel:

> "fast and intelligent."

---

# Responsive Rules

Desktop:

* full three-panel layout

Tablet:

* collapse right intelligence panel

Mobile:

* stacked narrative layout

Maintain narrative-first structure across all breakpoints.

---

# Accessibility Rules

Always support:

* keyboard navigation
* focus visibility
* readable contrast
* semantic HTML
* accessible touch targets

Minimum standard:
WCAG AA

---

# UI Non-Negotiables

NEVER:

* use generic admin templates
* create widget chaos
* overload dashboards
* prioritize charts over reasoning
* break spacing consistency
* create visual noise

ALWAYS:

* preserve calm UX
* prioritize AI narrative clarity
* maintain futuristic elegance
* keep layouts breathable
* reinforce institutional trust

---

# Final Principle

The UI should consistently feel like:

> "A billionaire investor's private AI intelligence workstation."

Users should feel:

* informed
* calm
* empowered
* intelligent
* in control
