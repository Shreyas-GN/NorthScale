# Frontend Rules

# Frontend Stack

Use:

* Next.js 15
* React 19
* TypeScript
* TailwindCSS
* shadcn/ui
* Zustand
* TanStack Query

---

# Frontend Philosophy

The frontend is:

* AI-first
* narrative-first
* calm
* premium
* responsive

NOT:

* widget-heavy
* chart-heavy
* spreadsheet-heavy

---

# Component Rules

## ALWAYS

* keep components small
* extract reusable UI
* use server-safe patterns
* strongly type props
* isolate presentation logic

---

## NEVER

* place business logic in components
* compute financial ratios in UI
* directly call scraping APIs
* hardcode recommendation logic

---

# State Management Rules

## Zustand

Use ONLY for:

* UI state
* sidebar state
* modal state
* preferences

---

## TanStack Query

Use for:

* server data
* stock analysis fetching
* caching
* async synchronization

---

# UI Rules

The interface must feel:

* futuristic
* layered
* cinematic
* intelligent
* readable

---

# Design Constraints

## NEVER

* use generic dashboard templates
* overload screens
* use bright neon gradients
* create noisy interfaces
* create cluttered tables

---

## ALWAYS

* preserve spacing consistency
* use subtle motion
* use layered surfaces
* maintain readable typography
* prioritize AI thesis visibility

---

# Animation Rules

Allowed:

* fades
* blur transitions
* hover elevation
* subtle scaling

Forbidden:

* bouncing
* flashy motion
* crypto-style effects

---

# Layout Rules

Global structure:

LEFT SIDEBAR
MAIN CONTENT
RIGHT INTELLIGENCE PANEL

Maintain this architecture consistently.

---

# Accessibility Rules

Always:

* keyboard navigable
* visible focus states
* accessible contrast
* semantic HTML

---

# Performance Rules

Optimize:

* rerenders
* hydration
* lazy loading
* data caching

Avoid:

* unnecessary client components
* oversized bundles
* blocking animations
